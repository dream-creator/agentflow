# Progressive Web App (PWA)

AgentFlow is installable as a PWA on iOS and Android. This
document covers the manifest, the service worker, and the
off-screen Turnstile iframe workaround.

## Manifest

`public/manifest.json`:

```json
{
  "name": "AgentFlow",
  "short_name": "AgentFlow",
  "description": "The CRM for agents who hate CRMs",
  "start_url": "/",
  "scope": "/",
  "display": "standalone",
  "theme_color": "#0F766E",
  "background_color": "#F0FDFA",
  "icons": [
    { "src": "/icons/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icons/icon-512.png", "sizes": "512x512", "type": "image/png" },
    { "src": "/icons/icon-512-maskable.png", "sizes": "512x512", "type": "image/png", "purpose": "maskable" },
    { "src": "/icons/icon.svg", "sizes": "any", "type": "image/svg+xml" }
  ]
}
```

- `display: standalone` — the app opens without the browser
  chrome.
- `theme_color` and `background_color` — teal palette to match
  the brand. The `background_color` is also used by iOS as the
  splash screen background.
- 4 icons — 192px and 512px PNG, 512px maskable (with safe-zone
  padding for adaptive icons on Android), and an SVG fallback
  for browsers that prefer it.

The manifest is linked from `src/app/layout.tsx` via the
`metadata.manifest` field.

## Icons

Files in `public/icons/`:

| File | Purpose |
| --- | --- |
| `icon-192.png` | Standard PWA icon (192x192). |
| `icon-512.png` | Standard PWA icon (512x512). |

The icon set uses a teal gradient with the AgentFlow wordmark.
Generated by the Figma design system.

## Service worker

`public/sw.js`. Strategy:

- **Network-first** for navigations, with cache fallback.
  This means the most recent HTML is served when online, but
  the app still loads from cache when offline (better than a
  blank page).
- **Cache-first** for static assets (`/_next/static/*`,
  `public/*`).
- **Skip** `/api/*` (always network).
- **Skip** cross-origin requests (no caching of third-party
  responses).
- **Skip** non-GET requests.

The cache name is `agentflow-v1`. To invalidate, bump the
version and re-deploy; old caches will be garbage-collected on
the next activation.

### Registration

`src/components/ui/sw-register.tsx` is a client component that
calls `navigator.serviceWorker.register("/sw.js")` from a
`useEffect`. Errors are silently caught (the registration can
fail in dev over HTTP, in browsers that don't support service
workers, or when ad-blockers are aggressive).

The component is mounted in the dashboard layout so the SW
activates on the first authenticated page view. This is
intentional — the landing page doesn't need offline support,
and avoiding the registration on `/` reduces a console warning
on Safari.

### Limitations

The current SW does not support:

- **Offline lead creation.** Forms in `/leads/new` won't submit
  when offline; they show the standard browser error.
- **Offline read.** The dashboard requires network to fetch
  leads from Supabase.

These are intentional — the app is online-first and adding
background sync would significantly complicate the data layer.
If offline support is needed, the recommended path is a
service worker that queues mutations in IndexedDB and replays
them on reconnect, paired with `BackgroundSync` API.

## Install prompt

The browser-native install prompt fires automatically on Chrome
desktop and Android when:

1. The page is served over HTTPS (or `localhost`).
2. The manifest is valid.
3. The service worker is registered and active.
4. The user has visited the site at least twice with a few
  minutes between visits (Chrome heuristic).

iOS Safari does not support the auto-prompt. Users install via
**Share → Add to Home Screen**. The `apple-touch-icon.png` and
`apple-mobile-web-app-capable` meta tag (set in
`src/app/layout.tsx`) make the installed icon and full-screen
launch work correctly.

## Off-screen Turnstile iframe

The Turnstile widget (a Cloudflare iframe) needs to be in the
DOM to verify the user, but its visible UI is undesirable on
the auth pages. The pattern in
`src/app/(auth)/login/page.tsx`:

```tsx
<div className="relative">
  {/* Visible form */}
  <form>...</form>
  {/* Hidden iframe */}
  <div className="absolute -left-[9999px] -top-[9999px] w-px h-px overflow-hidden pointer-events-none">
    <TurnstileWidget ... />
  </div>
</div>
```

The iframe is positioned off-screen with `absolute` and
`-left-[9999px]`. It's still rendered, still loads the
Cloudflare script, and still fires `onSuccess` — the user just
doesn't see it.

The `CaptchaStatusPill` component
(`src/components/auth/captcha-status-pill.tsx`) provides
visible feedback ("Verifying..." / "Verified") so the user
knows the captcha is doing something.

## Why a PWA and not native?

| | PWA | Native |
| --- | --- | --- |
| Distribution | Web (no App Store) | App Store review |
| Updates | Instant | App Store review |
| Push notifications | Limited on iOS | Full |
| Install friction | "Add to Home Screen" tap | Full App Store flow |
| Build effort | Already done | 2x codebase |

For a solo-agent CRM where the user base is small and
notifications are not yet a feature, PWA is the right trade.
